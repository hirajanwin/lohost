config.json

The config.json file should immediately be appear in the root of the lohost folder when lohost.exe is ran, if one doesn't already exist. It is populated with defaults that can be changed to configure your website.

This is a JSON file and so standard JSON formatting applies.

Remember, the lohost.exe will need to be stopped and started again for any config changes to be reflected.

Options

ApplicationId (Required)

This is the subdomain that the website will sit on lohost, https://{ApplicationId}.lohost.io. If the ApplicationId is already in use, the application will go into polling mode and check every 5 seconds if it now available.

Certain characters are not allowed as part of the ApplicationId: . ? / \ & #


ApplicationKey (Optional)

The ApplicationKey is optional but is needed to relate different running lohost.exe instances. If the ApplicationId and the ApplicationKey are the same it is assumed they are the same application.

This parameter is also used in conjunction with the ApplicationPaths parameter to map different URL routes to different lohost.exe instances.

The final and most important thing to know about the ApplicationKey is that it puts a temporary lock on the ApplicationId subdomain. If you stop lohost.exe, it will not allow another lohost.exe instance to take your subdomain without a matching ApplicationKey for 10 minutes.


ApplicationPaths (Optional)

This is an optional config option but at least one of your ApplicationId instances must be running with an empty ApplicationPaths variable (making it the default route).

This parameter is a string array and should correspond to a relative path within your "App" folder. Once defined, this will become where the website will serve the document from.

An example, if you have a folder stucture "App/route/mine", you can put an entry into the ApplicationPaths parameter "ApplicationPaths": [ "App/route/mine" ]. If someone then goes to the URL https://{ApplicationId}.lohost.io/App/route/mine/test.html, it will be served from that lohost instance.


IsListed (Required)

This config option determines if your website is listed on the lohost.io website, by default it is false.


Name (Optional)

This option is only used if the IsListed parameter is set to true. This is the name that your website will be listed as on the lohost.io website.


Tags (Optional)

This option is only used if the IsListed parameter is set to true. This is a JSON array that is used to tag your website on the lohost.io website.

An example of this would be, "Tags": [ "firstwebsite", "mine", "heyhey" ]


ApplicationFolder (Required)

This is the folder that your website will be hosted from. By default, this is "App" and it reall shouldn't be changed without reason.

If you have a burning desire to change it, it will work either via an absolute path, "MyWebsite", or a relative path "C:\MyWebsite"


LogsFolder (Required)

This is the folder where your website logs will be stored. By default, this is "Logs" and it reall shouldn't be changed without reason.

If you are a control freak and must change it, it will work either via an absolute path, "MyLogs", or a relative path "C:\MyLogs"


ExternalAPI (Required)

This is the API that the lohost client is connecting to in order to serve your website to the internet. This should ALWAYS be "https://lohost.io"

The only reason you would ever want to change this is if you are running your own instance of the API on your own endpoint. This will then also need to be changed for all of your lohost.exe configs.

Examples

A example for a default lohost.exe instance:

{
    "ApplicationId": "mytestsite",
    "ApplicationKey": "123",
    "IsListed": true,
    "Name": "My test site",
    "Tags": [ "firstwebsite", "mine", "heyhey" ],
    "ApplicationFolder": "App",
    "LogsFolder": "Logs",
    "ExternalAPI": "https://lohost.io",
    "ApplicationPaths": null
}


A example for a lohost.exe instance for mapping specific routes:

{
    "ApplicationId": "mytestsite",
    "ApplicationKey": "123",
    "IsListed": true,
    "Name": "My test site",
    "Tags": [ "firstwebsite", "mine", "heyhey" ],
    "ApplicationFolder": "App",
    "LogsFolder": "Logs",
    "ExternalAPI": "https://lohost.io",
    "ApplicationPaths": [ "App/route/mine" ]
}
